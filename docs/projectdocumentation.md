1. Problem Statement

    The objective of this assignment is to design and implement a modular, agentic automation system that converts raw product information into structured, machine-readable content pages.

    The system must:

    operate strictly on user-provided data

    avoid external knowledge or research

    generate content through a multi-agent architecture

    output clean JSON suitable for downstream systems

    The focus of the challenge is system design, agent orchestration, and modular automation, not UI or copywriting.

2. Solution Overview

    The solution is implemented as a multi-agent pipeline, where each agent performs a single, well-defined responsibility.

    The system workflow consists of:

    Parsing raw user input into structured data

    Validating and normalizing product information

    Generating categorized user questions

    Applying reusable content logic blocks

    Assembling pages using custom templates

    Writing final outputs as JSON files

    This design ensures extensibility, reusability, and clean separation of concerns.

3. Scope & Assumptions
    Scope

    The system processes one product at a time.

    Input is provided as raw text via the terminal.

    Output is strictly JSON (no UI or HTML).

    Generated pages:

    FAQ Page

    Product Page

    Comparison Page

    Assumptions

    Input follows the documented format.

    The comparison product is fictional and internally defined.

    No external APIs or datasets are used.

    4.2 Agent Responsibilities
        Product Parser Agent

        Converts parsed input into a validated product model.

        Ensures consistent internal representation.

        Question Generator Agent

        Automatically generates categorized user questions.

        Provides structured input for FAQ generation.

        Page Assembly Agent

        Orchestrates templates and logic blocks.

        Writes final JSON outputs.

        Agents do not directly depend on each other and communicate only via structured data.

        4.3 Content Logic Blocks

        Content logic blocks are reusable transformation units that derive structured content from product data.

    Examples:

        Benefits Logic Block

        Usage Logic Block

        Safety Logic Block

        FAQ Answer Logic Block

        Comparison Logic Block

        These blocks contain no orchestration or template logic.

    4.4 Template Engine

        Templates define the structure of each page type and assemble content by invoking logic blocks.

        Implemented templates:

        FAQ Template

        Product Page Template

        Comparison Page Template

        Templates are responsible only for composition, not content generation.

    4.5 Orchestration Model

        The system follows a controller-driven pipeline:

        Execution order is managed centrally.

        Agents are loosely coupled.

        Data flows in a single direction.

        This design ensures clarity, testability, and scalability.
